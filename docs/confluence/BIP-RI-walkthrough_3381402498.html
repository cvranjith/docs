BIP RI walkthrough

Created by Sougata Laga, last modified by Rahul Ajjarapu on Feb 17, 2022

Following are the steps incorporated for the ri of BI Publisher (BIP) :

(below commands to be run in terminal emulator environment)

  • sudo systemctl daemon-reload
    sudo systemctl restart docker
    sudo usermod -a  -G docker <NIS User name>  
  • sudo docker pull fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:5000/middleware/biplatform:12.2.1.4-06092021
  • BIP setup : sudo docker run --name fcubs-bip-119 -dit -p 9500:9500 -p 9502:9502 \
        -e ADMIN_USERNAME=weblogic \
        -e ADMIN_PASSWORD=weblogic123 \
        -e DB_HOST=100.76.142.253 \
        -e DB_PORT=8521 \
        -e DB_SERVICE=FCRPDB110 \
        -e DB_USERNAME=sys \
        -e DB_PASSWORD=Oracle123 \
        -e SCHEMA_PREFIX=RIBIP \
        -e SCHEMA_PASSWORD=Oracle123 \
    fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:5000/middleware/biplatform:12.2.1.4-06092021

          Wait till the command executes. Logs can be checked by : sudo docker logs <container id>/<container name> --follow

          Once this is complete, our BIP is up and running on the xmlp server. The server links are given as : 

                  a) Oracle WebLogic Server Administration Console        (username/pwd : weblogic/weblogic123)         

                  b) Oracle BI Publisher Enterprise Login

In the winscp, we may login to the bip server : http://fsgbu-mum-766.snbomprshared1.gbucdsint02bom.oraclevcn.com/ 

  • Copy xdo/xdm to server : Now, following command is to be given to push the required files related to report generation into the docker : 

docker cp /scratch/wls-docker/fcubs-bip-11-10/~administrator/CH220.xdm 107acef86bb8:/u01/oracle/user_projects/domains/bi/bidata/components/bipublisher/repository/Users/~administrator
docker cp /scratch/wls-docker/fcubs-bip-11-10/~administrator/CH220.xdo 107acef86bb8:/u01/oracle/user_projects/domains/bi/bidata/components/bipublisher/repository/Users/~administrator

Once this is done, we are ready for report generation. 

  • Datasource creation : Login to the xmlp server using credentials given. Then go to administration → JDBC connection → then add the required one, as shown below: (username/pwd: FCRUSR/FCRUSR)

Our jdbc connection is all set!

  • Report generation : Now in the homepage, go to Catalog → there we'll find the reports we want to generate. Click on it.


  • Following inputs are to be given in order to generate the reports :


  • Our report has been successfully generated!


  • Now we come to our next task in line. We need to lighten the bulk of entire bip container, so for that we have to remove some deployments from the weblogic. (except bipublisher 11.1.1. as this is our main application)
  • Login to weblogic console → deployments. 

After that, go to 'control' tab. Stop the deployments turn by turn and check if the reports are still getting generated or not. If yes, then we may proceed to remove (delete) those and activate any changes.

This is a trial and error basis as we don't exactly know which applications/deployments are indeed useful or not. 

In regular intervals, the bi-server has to be stopped and restarted to confirm whether the changes are getting reflected and we're obtaining desired results. After completing the process, following deployments remain : 


  • Now, we need to do the BIP FCR setup. For that, the FCR machine/server should use the BIP server's image for the report generation. 

So, login to this server: fsgbu-mum-764.snbomprshared1.gbucdsint02bom.oraclevcn.com and navigate through : /scratch/wls-docker/sfcubs-wls-14-5-1/user_projects/domains/base_domain/FCUBS144/Fcubs_property_file → fcubs.properties file and open it.

  • From here, we need to map the existing ip to the BIP's ip. So, locate the following snippet in the aforesaid properties file and make the necessary changes (highlighted in bold):

Make the changes in IP address and the port only.

####### Report PLUGIN PROPERTIES #######
BIP_END_POINT=http://100.76.142.253:9502/xmlpserver/services/v2/ReportService?wsdl
BIP_NAME_SPACE=http://xmlns.oracle.com/oxp/service/PublicReportService
BIP_ADV_SPOOL_PATH=D:\\ReportsSpool
BIP_USERNAME=y1jlGfFVGhsdnZiO8Gb0C8nvDaQ8pwhX
BIP_ADVICE_FOLDER=FCReports/Adv
BIP_PASSWORD=/VnqHPADQUviQNSVxXWn8jEqNQXIZm2W
BIP_SERVER_VERSION=12c
BRANCH_ADV_TYPE=NATIVE
BIP_TIME_ZONE=IST
BIP_REPORT_FOLDER=FCReports

(this is there in fcubs.properties inside location :
/scratch/wls-docker/sfcubs-wls-14-5-1/user_projects/domains/base_domain/FCUBS144/Fcubs_property_file)

  • Now the next step arises. This bip setup process (as mentioned before) has to be done in the machine where we want to run in for the FCR only. 

So, repeat the steps until bip is up and running in the desired machine. 

  • once done, stop the bi-server (this will take 2 mins approx)  and copy the setDomainEnv.sh into path /u01/oracle/user_projects/domains/bi/bin (file is attached below)

setDomainEnv.sh 

  • Start the bi-server again. This will take 3 mins approx. 
  • Datasource has to be created (as mentioned above)
  • one or two xdo & xdm have to be copied to the server, as mentioned above. 
  • Report generation is to be checked in biserver.
  • If successful, configure in the FCR server.
  • Check through the screen in FCR environment (FCR : 7775 --> CH220, view in 7778)
  • Once done, copy remaining xdo-xdm files and complete the setup fully.


Done. That's how the BIP-FCR setup can be accomplished.
