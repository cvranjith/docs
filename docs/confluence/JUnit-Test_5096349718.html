JUnit Test

Created by Sweta Chudasama, last modified on Oct 28, 2022

This document explains on how to write JUnit test cases for any java application.


JUnit is an open-source unit testing framework for Java Programming Language. It provides annotations to identify test methods. JUnit promotes the idea of "first testing then coding", which emphasizes on setting up the test data for a piece of code that can be tested first and then implemented.


First step is to create the Maven project and once the project is created, we can go to the "pom.xml" and add the dependency code for JUnit. Steps for the same are as follows:

  • Go to the following URL (Maven Repository: Central (mvnrepository.com)).
  • Search for JUnit in the search bar.

     

  • Page with various options for JUnit opens up. Click on JUnit.

     

  • Select the specific version (latest version recommended).

     


  • Copy the dependency code and paste it in pom.xml.

     


  • After dependency is added, it will automatically get installed and we can check that under Maven Dependencies.

Important: After adding dependency if any error persist in "pom.xml" then (Right Click on the project > Maven > Update Project > Check Force Update of Snapshots/Releases > Click on OK.

Once this is done, we will create a method to test the sum function and link it with the annotation "@Test" as follows:

 

In the above example we have created two variables actualValue which is calling the sum function by passing (10,20) as parameters to perform the addition and expectedValue as 30 since we know that addition of 10+20=30 this is done basically to test whether our function is working correctly or not. assertEquals is static method of Assert class which is provided by JUnit. assertEquals accepts two arguments "expected" and "actual" so in our case we have passed the two variable which we have declared. The significance of assertEquals is to check whether the two primitive/objects are equal or not.
