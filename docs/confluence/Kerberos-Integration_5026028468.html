Kerberos Integration

Process Flow


Technical Flow

  • Browser sends request to custom application ‘https://100.76.135.165:32009/kerberos-client/sso-login?userId=ADMINUSER3&entity=ENTITY_ID1’ after Kerberos validation with the required user details (User Id and entityID in our case).
  • The request is processed by the kerberosclient application.
  • kerberosclient generates a random token which along with the user info and timestamp is inserted into IFTB_CUSTOM_LOGIN table.
  • It then fetches the public key from server which is used to encrypt the generated token.
  • Finally all incoming request params are added to final url with encrypted token encrypted token under the ssoKey (KERBEROS in our case) and then it is redirected to login servlet of Oracle Banking application.
  • Once the redirect request is received at the Oracle Banking application side, the ssokey value is validated with the key value in fcubs.poperties and the token is passed to the custom AuthenticateCustomToken servlet.
  • Here the token is first decrypted with the private key available at server.
  • Then the decrypted token is used to fetch the user details from IFTB_CUSTOM_LOGIN table.
  • The time difference between record insertion and current time is compared against a standard set time limit.
  • If the comparison is passed then the record details are fetched and further passed along for login.
  • Finally the record is deleted from the IFTB_CUSTOM_LOGIN table.

Additional Configurations

Asymmetric Key generation:-

Use the following sample java application to create public and private key once or use the following publicKey and privateKey.

fcubs.properties :-

##### SINGLE SIGN ON PROPERTIES #####SSO_KEY=KERBEROS
SSO_REQ=Y
PASSWORD_EXTERNAL=N
SMS_SYMMETRIC_KEY=oraclefinancials
EXT_USERLOGIN=N
SSO_TYPE=TOKEN
KERBEROSKEYPATH=/u01/oracle/fsgbu/KeyPair/privateKey (should be same as path mentioned in custom app)
TOKENCUTOFFLIMIT=1000


kerberosclient :-

Maintain 'user-principal', 'keytab-location', 'access-url', 'jndi', 'pathToKey' and 'testMode' in application.yaml. 


Create table IFTB_CUSTOM_LOGIN as 

CREATE TABLE IFTB_CUSTOM_LOGIN(
ID VARCHAR2(36) NOT NULL, 
TOKEN VARCHAR2(72) NOT NULL, 
USER_ID VARCHAR2(100), 
LOGIN_DATE TIMESTAMP (6)
)
/
ALTER TABLE IFTB_CUSTOM_LOGIN
ADD CONSTRAINT PK_IFTB_CUSTOM_LOGIN PRIMARY KEY (ID) 
/
