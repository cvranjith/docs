Pushing an image to JAPAC Prod Registry

Created by Sougata Laga, last modified on Aug 26, 2022

This document is a quick guide to help push an image to our JAPAC prod registry.


Step-1 -  prepare your local image. (pull from dockerhub). For eg, let us take an example of this image: jenkins/jenkins:lts-jdk11



Step-2 : Tag that image to JAPAC Staging Repo



Step-3: Push the image to JAPAC Staging Repo


If we want to put any image to PROD registry (fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:5000), (UI is  http://fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:8021/), then → 

we first have to push to a staging registry which is : fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:6000



now the image should be available on remote repo (i.e. JAPAC Staging repo - i.e. 128.snbomprshared1.gbucdsint02bom.oraclevcn.com:6000).


We can also verify it using the URL: http://fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:8022/ (JAPAC PROD REGISTRY)


Step-4: Send mail to Liyakat

He will push to PROD registry (i.e. fsgbu-mum-128.snbomprshared1.gbucdsint02bom.oraclevcn.com:5000.)

once its pushed, he will confirm, and we will have to verify its there. 


NOTE: 128 is our PROD server.  We should not enter there, unless there is an emergency. all these commands should be done on our development VMs.



Hope this helps!
