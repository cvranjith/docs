Data Replication

Created by Sakshi Jain, last modified on Dec 01, 2022

Overview: This java webApp helps to replicate the data from an Oracle/non-oracle database to an Oracle database using JDBC connection and rest service.

Process flow:


Technical flow:

  • All the details of the source and destination database are maintained in application.yaml file.
  • On starting the webApp,
  • Then once we hit the endpoint "https://100.76.135.165:32009/replication-app/replication" where we can send the the required data.(entity, select statement).
  • This calls the async processMigration() method which will inturn call the processData() method of sourceImpl class. It will first create a JDBC connection to the source database and runs the select statement and selects data in the form of an JSONObject.
  • It then creates a JDBC connection to the destination database and calls the processData() method of destinationImpl class to run the procedure enclosed in the package. This procedure insert the data into the destination table.

  • Once these queries are run successfully, we can see the replicated data in the destination DB.


Configurations to be maintained:

application.yaml :-

database:
connections:
- type: source
url: "jdbc:oracle:thin:@100.76.136.97:1521:WING145" #### jdbc:vdb//100.76.136.97:1521/WING145
driver: "oracle.jdbc.OracleDriver" #####com.denodo.vdp.jdbc.Driver
jdbcProp:
username: BWING password: BWING
jndi:
- type: destination
url: "jdbc:oracle:thin:@100.76.136.97:1521:WING145"
driver: "oracle.jdbc.OracleDriver"
jdbcProp:
username: BWING
password: BWING
jndi:


Note: 1. URL, username and password should be updated in this file (URL format will be different for a non-oracle database).

2. Driver details should be updated in case of a non-oracle database.

3. The jndi details should be updated if the application is hosted on weblogic server.


ipfks_staging package :- This package has a procedure pr_process where you can add the insert statement just as below

create or replace package ifpks_replication
as

PROCEDURE pr_process(p_entityid in varchar2,p_req_msg in clob,p_rowcount in NUMBER,p_refno in varchar2,p_status in out varchar2);

end;
/
create or replace package body ifpks_replication
as

PROCEDURE pr_process(p_entityid in varchar2,p_req_msg in clob,p_rowcount in NUMBER,p_refno in varchar2,p_status in out varchar2) is
l_req json_object_t;
l_status VARCHAR2(100);
begin
l_req := json_object_t.parse(p_req_msg);
p_status:='Success';
insert into replication_dest values (p_entityid,p_req_msg,p_rowcount,p_refno,p_status);
end;

end;
